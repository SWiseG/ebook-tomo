# Parte 1 — O Padrão

## 1. Princípios fundamentais (não-negociáveis)

1. **Regra dos 3 segundos** — capa, título e primeira página têm 3 segundos para criar empolgação. Se falhar aqui, todo o resto é desperdício.
2. **Padrão Z de leitura** — o olho varre em Z (esquerda→direita, diagonal, esquerda→direita). Toda composição (capa, abertura de capítulo, CTA) respeita esse fluxo.
3. **Thumbnail Test** — toda capa permanece legível e atrativa reduzida a 150px. Se o título ou o impacto somem, refaça.
4. **Valor antes do pedido** — NUNCA CTA antes de entregar valor. O momento de ouro do CTA é o FIM de cada capítulo.
5. **Um capítulo = uma ideia** — uma ideia central, uma promessa cumprida, um micro-CTA. Capítulos de 800–1.500 palavras elevam a taxa de conclusão. Parágrafos com no máximo 4 linhas.

## 2. Sistema de tipografia

### Repositórios gratuitos (licença comercial)
Google Fonts · Fontshare · Font Squirrel · Velvetyne · The League of Moveable Type · Bunny Fonts · Open Foundry.

### Regra de ouro
- **Serifadas para o CORPO** (leitura longa, confiança): Merriweather, Lora, EB Garamond, Source Serif Pro, Literata, Georgia.
- **Sans-serif para TÍTULOS/dados/escaneabilidade**: Inter, Manrope, Plus Jakarta Sans, DM Sans, Archivo, Space Grotesk.
- **Display SÓ na capa e chamadas de impacto** (jamais no corpo): Bebas Neue, Anton, Playfair Display, Fraunces, Clash Display.

### Pareamento
- Sempre 2 fontes (máx. 3). Contraste visível obrigatório (peso/estilo).
- Combinação vencedora: serifa display no título + sans limpa no corpo, OU sans bold no título + serifa no corpo.
- Hierarquia: **H1 32–48 / H2 22–28 / H3 16–20 / corpo 11–13 / line-height 1.4–1.6**.

### Pareamento por nicho
| Nicho | Título | Corpo | Razão |
|---|---|---|---|
| Negócios/Finanças | Manrope Bold ou EB Garamond | Merriweather | Autoridade + leitura confortável |
| Self-help/Coaching | Fraunces | Plus Jakarta Sans | Calor humano + clareza |
| Tech/IA/SaaS | Space Grotesk | Inter | Modernidade tech |
| Saúde/Bem-estar | Lora | DM Sans | Acolhimento + leveza |
| Marketing/Vendas | Archivo Black | Inter | Impacto + escaneabilidade |
| Ficção/Romance | Playfair Display | Lora | Emoção + imersão |
| Educacional/Acadêmico | Source Serif Pro | Source Sans Pro | Credibilidade + clareza |
| Infantil/Criativo | Quicksand | Nunito | Amigável + arredondado |

## 3. Sistema de imagens (APIs gratuitas com licença comercial)

| Plataforma | Licença | Diferencial |
|---|---|---|
| Unsplash API | uso comercial OK, sem atribuição obrigatória | fotografia editorial premium |
| Pexels API | uso comercial livre | fotos + vídeos |
| Pixabay API | uso comercial livre | imagens, vetores, ilustrações, vídeos |
| Openverse / Wikimedia | Creative Commons | histórico, científico |
| NASA / Met Museum / Smithsonian | domínio público / CC0 | ciência, arte, artefatos |

Ícones/vetores: Phosphor, Heroicons, Lucide, Tabler, Iconoir (MIT/ISC); ilustrações: unDraw, Storyset, Humaaans.

### Regras de ouro
1. **≥ 1 imagem a cada 2–3 páginas** (forte componente visual eleva conclusão em ~30%).
2. **Coerência estilística** — não misturar foto realista + flat + 3D no mesmo e-book.
3. **Alta resolução** — mín. 1500px (impressão), 1200px (tela).
4. **Imagens trabalham** — nunca decorativas; cada uma prova um ponto ou cria pausa estratégica.
5. **Cumprir licenças** — manter log interno de origem mesmo quando a atribuição é opcional.
6. **Nunca clichê de stock** (aperto de mão corporativo, lâmpada de ideia, etc.).

## 4. Copywriting de alta conversão

- **AIDA** (macro do e-book): Attention (capa+título+1ª frase) → Interest (prefácio+sumário) → Desire (cases, transformações, prova social) → Action (CTA final + bônus).
- **PAS** (cada capítulo): Problem (nomear a dor) → Agitation (custo de não resolver) → Solution (método/insight).
- **4 U's** (todo título/subtítulo/CTA): Urgente, Útil, Ultra-específico, Único.
- **BAB** (intros e CTAs): Before → After → Bridge.
- **StoryBrand SB7** (marca): o leitor é o herói, o autor é o guia.

**7 gatilhos de Cialdini:** reciprocidade, compromisso/coerência, prova social, autoridade, afinidade, escassez, unidade.

**Fórmulas de headline:** "Como [X] sem [dor Y] em [tempo Z]" · "O método [adj.] que [resultado]" · "[N] [coisas] que [audiência] precisa saber sobre [tema]" · "A verdade que [autoridade] não quer que você saiba sobre [tema]" · "Por que [crença comum] está sabotando seu [objetivo]".

**Verbos de CTA:** Descubra, Domine, Conquiste, Acelere, Destrave, Multiplique, Transforme, Comece, Garanta, Acesse. **EVITAR:** clique, envie, submeta, registre-se.

## 5. Psicologia das cores

| Cor | Emoção | Nichos |
|---|---|---|
| Azul | Confiança, autoridade | Finanças, tech, saúde, B2B |
| Vermelho | Urgência, ação | Vendas, fitness, alerta |
| Verde | Crescimento, dinheiro | Bem-estar, finanças pessoais |
| Amarelo | Otimismo, atenção | Criatividade, infantil |
| Laranja | Entusiasmo, acessibilidade | Coaching, lifestyle |
| Roxo | Luxo, espiritualidade | Beleza, premium, esoterismo |
| Preto | Sofisticação, premium | Luxo, profissional, moda |
| Branco/Off-white | Limpeza, clareza | Minimalista, saúde, design |

**Regra 60–30–10:** 60% dominante + 30% secundária + 10% destaque (APENAS no CTA e pontos de atenção máxima).

## 6. Arquitetura do e-book que converte

Ordem obrigatória: (1) Capa · (2) Folha de rosto · (3) Copyright/ISBN · (4) Dedicatória (opcional) · (5) **Sumário visual** (ícones + 1 linha por capítulo) · (6) Prefácio/Carta ao leitor (BAB, fecha com promessa) · (7) **Capítulo 1: O Grande Insight** (valor massivo — ganha ou perde o leitor) · (8) Capítulos 2–N (hook → história/case → conteúdo → resumo visual → micro-CTA) · (9) **Capítulo final: O Próximo Passo** (síntese + CTA principal) · (10) Página de bônus · (11) Sobre o autor (por que VOCÊ) · (12) Contato/redes (QR codes).

**Elementos que elevam conversão:** pull quotes (a cada 3–4 págs), caixas "Insight Rápido" (fundo colorido), checklists ao fim de capítulos, infográficos, quadros "Estudo de Caso" (dados reais), espaços de anotação (workbook), CTA-respiro no meio do livro.

## 7. Workflow de entrega

- **Fase 1 — Diagnóstico:** nicho/tema, persona (idade, dor, consciência), objetivo de negócio, formato, tom de voz, identidade visual prévia, tamanho.
- **Fase 2 — Estratégia:** conceito + grande promessa; título + 3 alternativas; estrutura de capítulos com promessa de cada um; sistema visual (paleta hex + 2 fontes + estilo de imagem); CTAs mapeados.
- **Fase 3 — Produção:** capítulo por PAS; query exata de imagem (Unsplash/Pexels) por seção; elemento visual (pull quote/infográfico/checklist) por seção.
- **Fase 4 — Auditoria de conversão (checklist):**
  - [ ] Capa passa no Thumbnail Test?
  - [ ] Primeira página entrega valor em 60s de leitura?
  - [ ] Cada capítulo tem hook nos 2 primeiros parágrafos?
  - [ ] Prova social em ≥ 30% dos capítulos?
  - [ ] CTA final único, específico e urgente?
  - [ ] Tipografia respeita 32/24/18/12?
  - [ ] Imagem a cada 2–3 páginas?
  - [ ] Cores seguem 60–30–10?
  - [ ] Padrão Z nas aberturas de capítulo?

## 8. O que NUNCA fazer

Nunca: Comic Sans/Papyrus/Times New Roman default · mais de 3 fontes · CTA antes de entregar valor · aceitar "tema genérico" (force especificidade) · imagem clichê de stock · capa sem Thumbnail Test mental · jargão sem traduzir · terminar capítulo sem micro-fechamento + ponte · parágrafo com mais de 4 linhas · mais de 1 ideia por capítulo.

# Parte 3 — Plano de melhoria (5 frentes, track paralelo às Ondas)

### ✅ Frente A — Tipografia + cor profissionais *(CONCLUÍDA 2026-06-18)*
- **6 fontes profissionais embarcadas** (Inter, Manrope, Merriweather, Lora, Fraunces, Playfair Display) — estáticos Regular+Bold instanciados de fontes variáveis Google, em `src/Ebook.Api/assets/fonts/`.
- `FontRegistry` registra no QuestPDF (`RegisterFontWithCustomName`, família derivada do nome do arquivo) e no Skia (cache por família+peso, `FromFile`); fallback gracioso.
- `NicheStyleCatalog`: hash → **semântico por nicho** (cores emocionais + par de fontes da tabela acima). `PaletteCatalog`/`PdfThemeSelector` delegam a ele.
- `QuestPdfRenderer`: cores/fontes da paleta do nicho (coerente com a capa), hierarquia 40/26/18/12 + line-height 1.5, accent reservado a destaques (60-30-10).
- **Verificado**: PDF do nicho Finanças embarca `Manrope-Bold`/`Merriweather-Regular`, **sem fallback para Lato**. 154 testes OK.

### ✅ Frente B — Prompts de conteúdo (AIDA/PAS/estrutura) *(CONCLUÍDA 2026-06-18)*
- `outline.md`: macro-AIDA (Cap 1 "O Grande Insight", Cap final "O Próximo Passo"), 1 ideia/capítulo (800–1.500 palavras), título com 4 U's.
- `chapter.md`: PAS, hook em 2 parágrafos, parág. ≤4 linhas, pull quote (`>`) + checklist (`- `), micro-CTA com verbo de comando.
- `sales-copy.md`: 4 U's, gatilhos de Cialdini, BAB, fórmulas de headline, verbos de conversão (schema JSON preservado).

### ✅ Frente C — Layout rico no PDF *(CONCLUÍDA 2026-06-18)*
- `MarkdownBlockKind` estendido (`PullQuote`, `Callout`) + parser reconhece `> frase` (pull quote), `> [!INSIGHT]`/`> [!CASO]` (caixas) e `- [ ]`/`- [x]` (checklist).
- `QuestPdfRenderer`: pull quote (barra accent + itálico), caixa de destaque (borda accent + fundo claro + rótulo), checklist com checkbox desenhado (sem depender de glifo).
- `chapter.md` emite os marcadores. 156 testes OK.
- **Pendente (próxima leva):** bloco de imagem no corpo (vem com a Frente D/E14), CTA-respiro no meio, sumário visual com ícones (Phosphor/Lucide).

### 🔵 Frente D — Imagens no corpo *(monta sobre a Onda 2 — E14 Media Gateway)*
- 1 imagem / 2-3 páginas, de: E14 (geração free-first) para conceitos + Unsplash/Pixabay (hoje só Pexels) para foto editorial.
- A `query` por seção (Frente B) alimenta o gateway; o `PdfJobHandler` injeta as imagens.

### 🔵 Frente E — Auditoria de conversão *(liga ao E16 + E15)*
- Passada de IA validando o checklist da Fase 4 como **gate antes de publicar** — o "linter de persuasão" do E16 aplicado ao e-book.
